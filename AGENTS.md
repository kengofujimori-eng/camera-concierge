# AGENTS.md

## Project
Camera Concierge is a Next.js 14 app for camera and lens recommendations.
It uses Dify/Gemini for chat responses and local lens DB data for recommendation cards, images, prices, mount filtering, and warehouse behavior.

## Core principles
- Keep changes small and PR-sized.
- Do not change recommendation logic unless explicitly requested.
- Do not change API / Dify behavior unless explicitly requested.
- Do not change localStorage formats unless explicitly requested.
- Preserve existing E2E hooks and data-testid values.
- Prefer safe, incremental fixes over broad rewrites.

## UI direction
- Current design direction: Precision Console.
- Use white, slate, and neutral surfaces as the base.
- Use indigo / blue / violet / magenta gradients only as local accents or glow.
- Do not add full-screen purple haze or heavy glass backgrounds.
- Prefer thin borders, clear hierarchy, readable text, and product UI feel.
- Scene icons should use lucide-react rather than large emoji.
- Keep sidebar readable in both light and dark modes.

## Lens data rules
- Main data file: public/lens_data.json.
- Do not reformat the whole JSON file.
- Keep diffs minimal.
- Do not change image URLs, price_info, purchase_links, recommendation_status, or availability_status unless the task asks.
- Prefer current/new-generation lenses over discontinued old generations in normal recommendations.
- Mark old, ambiguous, mixed-mount, or adapter-era records as caution or avoid when appropriate.
- For ambiguous multi-mount records, do not guess. Report or quarantine them.

## Image audit rules
- Contact sheets are generated by npm run audit:images.
- audit-output/ is generated output and must not be committed.
- If changing lens images, regenerate processed images and run audit:images.
- Do not use a different product image when the exact lens cannot be verified.

## Testing
For normal code, UI, or data changes, run:
- npm run build
- npm run db:check
- npm run test:e2e

For image/data audit changes, also run:
- npm run audit:images

If Playwright/Chromium fails because of the local environment, report the exact error and do not claim E2E passed.

## E2E selectors
Never remove or rename existing data-testid values unless explicitly requested:
- selected-mount-display
- mount-button-*
- mount-option-*
- chat-input
- chat-send-button
- assistant-answer
- lens-card
- lens-card-image
- lens-card-placeholder
- price-badge

## Git rules
- Make one focused commit per task.
- Do not create PRs unless requested.
- Report changed files, checks run, git status --short, and git log --oneline -3.
- Do not commit generated audit-output files.
- Do not commit .env files or secrets.

## Commit prefixes
- style: UI-only changes
- data: lens_data.json metadata changes
- fix: bug fixes
- test: E2E or test changes
- ci: GitHub Actions changes
- tool: scripts or audit tools
- chore: cleanup / workflow
- docs: documentation

## Before finishing
Always confirm:
- working tree is clean
- relevant checks passed
- no unintended files are included

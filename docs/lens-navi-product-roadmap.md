# Lens Navi Product Roadmap

## 1. 基本方針

Lens Navi の本筋は、撮影前の意思決定を助けるアプリである。

ユーザーはまず「何を撮りたいか」から入り、そこから必要な焦点距離、F値、レンズ構成を理解する。レンズ名や価格だけを先に並べるのではなく、撮影シーンと条件から「なぜその候補が自然なのか」を納得できることを重視する。

倉庫、ギャラリー、カメラバッグは、シーンガイドとAI深掘りの後段に置く。所有レンズを管理したり眺めたりする体験は重要だが、Lens Navi の中心価値は「撮る前の判断」を支えることにある。

## 2. ロードマップ

今後は以下の順番を基本線として進める。

1. シーンガイド完成
2. AI深掘り完成
3. ギャラリー追加
4. カメラバッグ機能追加
5. AIカメラバッグアドバイザー化

この順番は固定の開発日程ではなく、プロダクト上の優先順位である。公開βの利用状況や重複フィードバックによって前後する可能性はあるが、ギャラリーやカメラバッグを本筋より先に大きく広げすぎない。

## 3. Phase 1: シーンガイド完成

シーンガイドは Lens Navi の入口であり、「何を撮るか」から必要なレンズ条件へ進むための中心機能である。

主な到達目標:

- 主要シーンの完成
- 文字量の圧縮
- 焦点距離理解の視覚化
- 初心者が迷わず使える状態にする

Scene Guide はすでに主要4シーンで interactive 化されているが、公開βの実利用を見ながら、説明量、条件選択の分かりやすさ、焦点距離の見え方をさらに整える。

## 4. Phase 2: AI深掘り完成

AI深掘りは、シーンガイドで出た判断を「納得できる説明」に変える段階である。

主な到達目標:

- 推薦理由の深掘り
- 比較理由
- なぜこの焦点距離 / F値なのか
- シーンごとのレンズ構成理解
- 初心者にも納得できる説明

ここでは、単に候補レンズを増やすのではなく、ユーザーが「自分の撮影条件では何を優先すべきか」を理解できることを重視する。

## 5. Phase 3: ギャラリー追加

`/warehouse-gallery` は本筋UIではなく、所有レンズを愛でるギャラリーモードとして扱う。

黒いマットブラック / オニキスの世界観は、Lens Navi の標準白ベースUIとは異なるが、所有レンズをコレクションとして眺める特別な鑑賞モードとして有効である。そのため、完全に捨てるのではなく、標準倉庫とは役割を分けて残す。

重要な位置づけ:

- `/warehouse-gallery` は `/warehouse` の実用管理画面を即置換するものではない
- 標準の倉庫UIは、保存、所有 / 欲しい状態、実用管理を担う
- ギャラリーは、所有レンズのコレクションビューまたは表示モードとして扱う
- 将来的には倉庫内の表示モードとして統合する可能性がある

## 6. Phase 4: カメラバッグ MVP

カメラバッグは、イベント別 equipment list として設計する。

想定イベント:

- ハワイ旅行
- 運動会
- 発表会
- 家族旅行
- 七五三
- 仕事撮影

ユーザーはイベントごとに、持ち出す機材を選定・保存できる。

MVPで想定する情報:

- イベント名
- 目的メモ
- 所有レンズから持ち出し選択
- 構成メモ
- 焦点距離カバレッジ
- 重量合計
- 不足している役割の簡易表示

カメラバッグは、所有レンズ管理とシーンガイドが自然につながった後に着手する。先に作り込むと、単なる持ち物リストになり、Lens Navi の判断支援価値から離れやすい。

## 7. Phase 5: AI Camera Bag Advisor

AI Camera Bag Advisor は、シーンガイドと所有レンズを接続する段階である。

イベント内容から、おすすめ持ち出し構成を提案する。

例:

- 次のハワイ旅行にどのレンズを持っていくか
- 運動会で望遠をどうするか
- 発表会で明るさと焦点距離をどう確保するか

所有レンズだけで足りない場合は、wishlist や購入候補へ接続する。ここでも「絶対のおすすめ」ではなく、撮影条件と所有機材に対して自然な候補を示す。

## 8. `/warehouse-gallery` の再定義

旧方針では、`/warehouse-gallery` を `/warehouse` の統合候補として見ていた。

今後は、完全置換前提ではなく「ギャラリーモード」として扱う。

理由:

- 単体ビジュアルは強いが、Lens Navi 本体の白ベース・相談型UIとはトーンが異なる
- 本筋UIにそのまま統合すると別アプリ感が出る
- 一方で、所有レンズを愛でる体験としては非常に有効

したがって、黒いギャラリーは継続するが、標準倉庫UIとは役割を分ける。ギャラリーは「所有レンズの鑑賞」「コレクションの満足感」「詳細へ進む深掘り導線」を担い、標準倉庫は実用管理を担う。

## 9. 直近の優先順位

まずはシーンガイドとAI深掘りを完成させる。

ギャラリーは並行実験として継続してよいが、本筋優先度はシーンガイド / AI深掘りを上位に置く。カメラバッグは、シーンガイドと所有レンズ管理がつながった後に着手する。

直近の判断基準:

- Scene Guide の公開βフィードバックを優先して見る
- AI相談の説明品質に重大な問題があれば、AI深掘りを優先する
- `/warehouse-gallery` は所有レンズを愛でる表示モードとして磨く
- `/warehouse` の localStorage、本番データ構造、route 統合は別途レビューしてから触る
- カメラバッグは、所有レンズとシーン判断の接続が見えてから最小MVPを検討する

## Guardrails

- `/warehouse-gallery` を `/warehouse` の即時置換として扱わない
- ギャラリーの黒い世界観を Lens Navi 全体の標準UIへ広げない
- localStorage、route、本番データ構造はロードマップ整理だけで変更しない
- stable_id や Deep Review 接続を急がない
- API / Dify / public data をロードマップ整理と同時に変更しない
- スコア、ランキング、点数表現に寄せない

# Warehouse Gallery (武器庫ギャラリー) — 進捗と次ステップ

倉庫UIを「データベース感のある一覧」から「レンズを愛でるコレクション体験」へ
刷新するための、世界観プロトタイプの記録。実験ページ
`src/app/warehouse-gallery/page.tsx`（既存 `/warehouse` とは独立）で検証中。

## 目指す世界観
マトリックスの仮想武器庫のように、呼び出すとレンズが整然と現れる。
近未来的・ミニマル・高級。マットブラック/オニキス。データベース感を消し、
レンズそのものを主役に、文字情報は最小、選択して初めて詳細が開く。
参照: Apple製品ページ、Air（awwwards）、Sigmaレンズ一覧、Armory、THE UNFAZED。

## 次回やること（v9 以降）

### 1. 背景を戻す
- gallery-bg-circles.mp4（同心円）は明度差が出て主張が強くうるさい。
  → 前の gallery-bg.mp4（幾何学ファセット、均一で静か）に戻す。

### 2. 二段階目（詳細パネル）の調整
- 「詳しく見る」トリガーが控えめすぎて気づかない → もう少し目立たせる。
- 「新品で買う」ボタンのグラデが目立ちすぎ＋左端の色が不自然 → 品よく抑える。
- 詳細パネルの中を「文字タブ」で再構成（アイコン化は構造確定後の将来対応）。

### 3. タブ構成（本番データ構造に基づく設計）
lens_data.json の1レンズが持つ実フィールドに紐づけて、3タブで構成:
- **AI解析タブ**: `recommendation_note`（解説文）＋ `recommendation_status`
  （recommend/caution/avoid をバッジ表示）。実データあり、すぐ本物にできる。
- **価格・購入タブ**: `price_info`（new_price/used_price）＋ `purchase_links`
  （new: amazon/rakuten/yahoo、used: kitamura）。複数リンクを品よく並べる設計が要る。
- **レビュー・作例タブ**: `review_links`（現在は空 {}）。本番で複数入る前提。
  ここはさらに細分化を検討（プロ評価/海外レビュー/作例ギャラリー等）。器を先に作る。
- 参照系: `official_url`（メーカー公式）等は補助的に。

### 4. レビュータブの細分化（要検討）
- review_links に入る項目が多くなる想定。レビュータブ内でさらに小分けする構造を設計。

### 5. 将来: 本番への統合
- 実験ページ /warehouse-gallery の世界観を、本番倉庫 /warehouse に統合。
- 実データ接続（lens_data.json）、所有/欲しいの永続化、FocalMap や
  価格履歴・買い時シグナル（②）との結合。
- アイコン化（THE UNFAZED 風：アイコンで情報切替し文字密度を下げる）は
  タブ構造が固まってから。

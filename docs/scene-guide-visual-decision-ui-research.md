# Scene Guide visual decision UI research

## Purpose

Lens Navi の Scene Guide を、長文の読み物ではなく、撮影条件を直感的に選びながら必要なレンズ条件へ到達する UI に発展させるためのリサーチメモ。

目的は、複雑なレンズ選びを以下の流れへ圧縮すること。

撮影条件を選ぶ
↓
必要な焦点距離・明るさ・ズーム安全性が見える
↓
候補レンズ条件が絞られる
↓
相談 / AI深掘りへ引き継ぐ

Lens Navi はレンズレビューサイトではなく「撮影判断ナビ」である。  
そのため、Scene Guide は文章量で説明するのではなく、条件選択と視覚的フィードバックで判断を補助する。

## Problem

現在の Scene Guide は、家族写真で interactive decision flow が導入され、発表会 / 運動会も detail 表示を持つ。

しかし、発表会や運動会では情報量が多く、以下の問題が残る。

- 焦点距離、明るさ、距離、会場サイズ、被写体の動き、ズーム安全性が同時に出る
- 色分けだけでは情報が整理されない
- 読む量が増えると、ガイドではなく記事に近づく
- 条件を変えると候補が変わる体感がまだ弱い
- 最終的に「何を相談へ渡せばよいか」が見えにくい

## Design principles

複雑な選択を扱う成功UIに共通する原則。

### 1. Progressive disclosure

最初からすべてを見せない。  
必要なタイミングで、必要な条件だけを開く。

Lens Navi では、最初に見せる条件を3つ程度に絞る。

発表会:
- 座席位置
- 会場サイズ
- 狙い

補助条件は後から開く。

詳細条件:
- レンズ交換できるか
- 動画も撮るか

### 2. 3択以内の選択

ユーザーは専門用語を理解したいのではなく、今の撮影条件を選びたい。

悪い例:
- 85mm
- 100mm
- 135mm
- 200mm
- 70-200mm
- 100-400mm

良い例:
座席は？
- 前方
- 中央
- 後方

### 3. 即時フィードバック

条件を選んだ瞬間に、結果が変わる必要がある。

例:
- 中央席 + 小ホール + 表情 -> 135mm が強くなる
- 後方席 + 大ホール + 表情 -> 200mm+ が強くなる
- 動画も撮る -> 70-200mm の安全性が上がる

### 4. 役割別の結果表示

絶対的な1位を出さない。  
撮影条件に対して、役割で出す。

例:
- 本命: 135mm
- 安全策: 70-200mm
- 条件付き: 85mm

### 5. グラフィックは装飾ではなく条件の可視化

Lens Navi では派手なイラストより、抽象的な条件メーターや焦点距離レールが合う。

例:
- 距離: 近い ---- 遠い
- 会場: 狭い ---- 広い
- 明るさ: 明るい ---- 暗い
- 柔軟性: 単焦点 ---- ズーム安全

## Visual UI patterns for Lens Navi

### A. Focal length rail

焦点距離の位置関係を直感化する。

35mm - 50mm - 85mm - 135mm - 200mm+

状態例:

家族写真 / 室内:
35mm / 50mm が濃くなる

発表会 / 中央席:
135mm / 70-200mm が濃くなる

運動会 / 広い校庭:
200mm+ が濃くなる

目的:
- 数字を読ませず、距離感を見せる
- 条件変更で候補レンジが動くことを体感させる
- レンズ名より先に「必要な距離」を理解させる

### B. Condition meters

撮影条件の重さを視覚化する。

例:
- 距離: 近い -> 遠い
- 会場: 狭い -> 広い
- 明るさ: 明るい -> 暗い
- 柔軟性: 単焦点 -> ズーム
- 動き: 少ない -> 多い

目的:
- 条件の意味を短時間で理解できる
- 色を増やさず、位置と濃淡で伝える
- 文章量を減らせる

### C. Candidate role cards

候補をランキングではなく役割で見せる。

例:
- 本命: 135mm
- 安全策: 70-200mm
- 条件付き: 85mm

目的:
- 「どれが一番か」ではなく「どんな役割か」を伝える
- ユーザーの迷いを残したまま整理する
- AI深掘りへの接続を自然にする

### D. Minimal reason line

結果の説明は1行に抑える。

例:
中央席で表情を切り出すには、85mmでは短く、135mmが自然です。

詳細説明は Deep Review / AI深掘りへ逃がす。

## Scene-specific UI directions

### 家族写真

Primary question:
- 室内か屋外か

Visual result:
- 室内 -> 35mm / 50mm
- 屋外 -> 85mm / 135mm

Additional condition candidates:
- 子どもが近づいてくる
- 複数人を撮る
- 背景を残したい

### 発表会

Primary questions:
- 座席位置
- 会場サイズ
- 狙い

Visual result:
- 前方 / 小ホール / 全身 -> 85mm
- 中央 / 小ホール / 表情 -> 135mm
- 席不明 / 動画あり -> 70-200mm
- 後方 / 大ホール / 表情 -> 200mm+

### 運動会

Primary questions:
- 会場の広さ
- 子どもまでの距離
- 動きの速さ

Visual result:
- 近い競技 -> 70-200mm
- 広い校庭 -> 100-400mm / 200mm+
- 待機中の表情 -> 85mm / 135mm

## UI tone

- 白 / slate ベース
- thin border
- restrained shadow
- blue-violet-magenta は局所アクセント
- large gradient fill は使わない
- noisy neon は避ける
- スコア / ランキング表現は避ける

## Recommended implementation order

1. 発表会ガイドを interactive 化
2. 運動会ガイドを interactive 化
3. Scene Guide の結果を「相談に引き継ぐ」導線へ接続
4. Lens condition resolver を設計・実装
5. AI深掘り比較へ接続

## Guardrails

- 1画面に全部出さない
- 選択肢を増やしすぎない
- グラフィックは装飾ではなく判断補助に使う
- レンズ名より先に条件を出す
- 最終出力はレンズ名ではなく、必要条件と候補役割にする
